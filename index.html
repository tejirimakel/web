Media Query

The CSS Media Query gives you a way to apply CSS only when the browser and device environment matches a rule that you specify, for example "viewport is wider than 480 pixels".

Syntax:

A simple media query looks like :

@media media-type and (media-feature-rule) {
/*CSS rules go here*/
}

It consists of:

  • A media type, which tells the browser what kind of media this code is for (e.g. print, or screen).
  • A media expression, which is a rule, or test that must be passed for the contained CSS to be applied.
  • A set of CSS rules that will be applied if the test passes and the media type is correct.

Media Type:

Possible media type are:

  • all
  • print
  • screen

Example:

@media print{
  body {
   font-size: 10px;
  }
}

Above media query will change font-size of the body only while printing the page.

Media Feature Rule:

After specifying the media type you can target the media features

Width & Height

The most common feature used to create responsive design is the viewport width. We can apply CSS rules if the viewport with is above or below a certain width or an exact width using min-width, max-width, width media features.

@media screen and (width: 600px) {
  body {
   color: red;
  }
}

Above rule will apply only when the viewport with is 600px and media type is screen.

@media screen and (max-width: 599px){
  body{
   color: red;
  }
}

Above rule will apply when viewport is 599px or smaller.

@media screen and (min-width: 700px){
  body{
   color: bisque;
  }
}

Above rule will apply when viewport is 700px or larger.

In practise width and height are not used frequently.

Orientation

Orientation media feature allow us to test for portrait or landscape mode.

@media screen and (orientation: landscape){
  body{
   color: red;
  }
}

Above rule will apply when device orientation is landscape. Standard desktop view has landscape orientation.

Using range syntax

One common case is to check if the viewport width is between two values:

@media (min-width: 30em) and (max-width: 50em) {
/*CSS rules go here*/
}

OR

@media (30em <= width <= 50em) {
/*CSS rules go here*/
}

Breakpoints:

There are tons of screens and devices with different heights and widths, so it is hard to create an exact breakpoint for each device. To keep things simple you could target five common groups:

/* Extra small devices (phones, 600px and down) */
@media only screen and (max-width: 600px) {...}

/* Small devices (portrait tablets and large phones, 600px and up) */
@media only screen and (min-width: 600px) {...}

/* Medium devices (landscape tablets, 768px and up) */
@media only screen and (min-width: 768px) {...}

/* Large devices (laptops/desktops, 992px and up) */
@media only screen and (min-width: 992px) {...}

/* Extra large devices (large laptops and desktops, 1200px and up) */
@media only screen and (min-width: 1200px) {...}
